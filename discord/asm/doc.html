Help and introduction

The bot uses mention prefix.

@Assembler help Displays help.

Server management

/guild rename <name...> Renames current server. This command requires manage server permission.
/guild info Displays information about current server. This command can be used by anyone.
/guild config prefix <prefix> Changes command prefix for current server. Setting it to "" will reset it to default. This command requires manage server permission.
/guild config modlog <channel> Configures moderation log for current server. This command requires manage server permission.
/guild config muterole <role> Configures mute role for current server. This command requires manage server permission.

Member management

/members report <member> <reason...> Reports a member to moderators. This command requires moderation log to be configured on the server (see /guild config modlog), and can be used by anyone.
/members mute <member> [duration] Mutes a member for a specified time. If duration is not specified, mute is permanent, and can only be lifted manually. This command requires configured mute role (see /guild config muterole), as well as moderation log (see /guild configure modlog), as well as kick members permission.
/members unmute <member> Unmutes a member. This command requires configured mute role (see /guild config muterole), as well as moderation log (see /guild configure modlog), as well as kick members permission.
/members kick <member> Kicks a member. This command requires moderation log to be configured on the server (see /guild config modlog), as well as kick members permission.
/members softban <member> Softbans a member. Softban is a ban and instant unban. Functionally, it works like kicking, except, since a ban occurs, it also removes messages of the kicked member. The member is free to rejoin once softbanned. This command requires moderation log to be configured on the server (see /guild config modlog), as well as kick members permission.
/members ban <member> [duration] Bans a member for a specified time. If duration is not specified, ban is permanent, and can only be lifted manually. This command requires moderation log to be configured on the server (see /guild config modlog), as well as ban members permission.
/members unban <member id> Unbans a member by his user ID. This command requires moderation log to be configured on the server (see /guild config modlog), as well as ban members permission.
/members prune [days] Prunes inactive members. Inactive members are ones that have not logged in within specified amount of days (default of 30), and were not assigned a role. The maximum number of days you can specify is 30. This command requires kick members permission.
/members information Displays information about a member. This command can be used by anyone.
/members reason <case id> <reason...> Sets a moderator action's reason, as well as responsible moderator. This command requires configured mute role (see /guild config muterole), as well as moderation log (see /guild configure modlog), as well as kick members and ban members permissions.

Role management

/roles create <name...> Creates a new role with specified name. This command requires manage roles permission.
/roles delete <role> Deletes specified role. This command requires manage roles permission.
/roles edit <role> <parameters...> Edits specified role. Arguments are passed as argument1=value1 argument2=value2. Possible arguments are name permissions colour hoist mentionable. This command requires manage roles permission.
/roles information <role> Displays information about specified role. This command requires manage roles permission.
/roles list Lists roles defined in specified server. This command requires manage roles permission.
/roles adduser <member> <roles...> Adds specified member to specified roles. This command requires manage roles permission.
/roles removeuser <member> <roles...> Removes specified member from specified roles. This command requires manage roles permission.

Channel management

/channels purge <channel> [count] Purges specified channel. If count is not specified, it defaults to 100. This command requires manage channels permission.

Bot control

/bot nickname <nickname...> Changes bot's nickname. This will make the bot nickname itself PAM Bot (specified nick). If nickname is set to "", it will reset the nickname. This command requires manage nicknames permission.
/bot about Displays information about the bot. This command can be used by anyone.
/bot invitelink Displays the link to invite this bot to your own server. This command can be used by anyone.

RSS management

/rss add <channel> <url> [tag] Configures an RSS feed in specified channel. If tag is specified, it will be appended to posts' titles as [tag]. This command requires manage server permission.
/rss remove <channel> <url> [tag] Removes an RSS feed from specified channel. This command requires all arguments to match those used to create the feed. This command requires manage server permission.
/rss list Lists all feeds configured in current server. This command requires manage server permission.

Tag management

/tag <name...> Displays a tag. This command is type-agnostic, meaning it will work with channel-specific, server-specific, and global tags. It will pick a best match (i.e. channel tag if one exists, falls back to server-wide, then to global). This command can be used by anyone.
/tag create <name> <contents...> Creates a new tag with specified name and contents. The created tag will only be accessible in the channel it was created in. This command requires manage messages permission.
/tag create guild <name> <contents...> Creates a new tag with specified name and contents. The created tag will be accessible from each channel of the server it was created in. This command requires manage server permission.
/tag edit <name> <new contents...> Edits a tag with specified name, and replaces its contents. This command can only be used to edit tags created in the channel it is being invoked in. This command requires manage messages permission.
/tag edit guild <name> <new contents...> Edits a tag with specified name, and replaces its contents. This command can only be used to edit server-wide tags. This command requires manage server permission.
/tag delete <name> Deletes a tag with specified name. This command can only be used to delete tags created in the channel it is being invoked in. This command requires manage messages permission.
/tag delete guild <name> Deletes a tag with specified name. This command can only be used to delete server-wide tags. This command requires manage server permission.
/tag info <name> Displays information of a tag with specified name. This command is type-agnostic, meaning it will work with channel-specific, server-specific, and global tags. This command can be used by anyone.
/tag info guild <name> Displays information of a tag with specified name. This command can only be used to display information of server-wide tags. This command can be used by anyone.
/tag info global <name> Displays information of a tag with specified name. This command can only be used to display information of global tags. This command can be used by anyone.
/tag dump <name> Dumps the contents of a tag with specified name. This command can only be used to dump tags created in the channel it is being invoked in. This command requires manage messages permission.
/tag dump guild <name> Dumps the contents of a tag with specified name. This command can only be used to dump server-wide tags. This command requires manage server permission.
/tag list Lists tags available from the channel the command was invoked in. This command can be used by anyone.

GNU/Stallman module management

This module responds with the famous Richard Stallman copypasta whenever someone says linux, but not gnu/linux. It is enabled by default.

/stallman disable Disables the module. This command requires manage server permission.
/stallman enable Enable the module. This command requires manage server permission.

Fun commands

This module has 2 hidden commands. Can you figure them out?

/fun random [min] [max] [count] Generates a random number between min and max, which default to 0 and 32768, respectively. The count parameter specifies how many numbers to generate, and defaults to 1. This command can be used by anyone.
/fun dice [count] Generates a random number between 1 and 6. The count parameter specifies how many numbers to generate, and defaults to 1. This command can be used by anyone.
/fun choice <choices...> Picks a random option from the ones supplied. This command can be used by anyone.
/fun markov channel [channel] Generates a markov chain from up to 200 messages from a specified channel. If channel is not specified, it will default to current one. This command can be used by anyone.
/fun markov user <member> [channel] Generates a markov chain from up to 800 messages posted by a specified member, in a specified channel. If channel is not specified, it will default to current one. This command can be used by anyone.
/fun markov role <role> [channel] Generates a markov chain from up to 400 messages posted by a members of specified role, in a specified channel. If channel is not specified, it will default to current one. This command can be used by anyone.
/fun colour <colour> Generates a 96x96 square of specified colour. The colour must be an RRGGBB or an AARRGGBB hex string. This command can be used by anyone.
